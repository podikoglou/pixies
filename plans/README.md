## Findings considered and rejected

- **Static-file path traversal in the SPA fallback** (`packages/server/src/index.ts:459-467`): not worth fixing because the URL constructor resolves `..` dot-segments before `path.join`, so `/../../etc/passwd` normalises to `/etc/passwd` and joins inside `webDist`. Verified empirically against both node and bun. Re-audit only if the static-serving code is rewritten to bypass `new URL(...)`.
- **Monty code-execution sandbox**: WASM Python with no import/FS/network surface — contained by design, not a finding.
- **X-Forwarded-For spoofing** (`packages/server/src/rate-limit.ts:142-163`): honours the standard rightmost-trusted-hops proxy convention; by-design, not a finding. (Prior issues #100/#222 already hardened this path.)
- **Fire-and-forget DB writes** in `ConversationStore.create`/`streamPrompt`/`delete` (`packages/server/src/conversations.ts`): pervasive but by-design best-effort persistence for a demo; failures are logged. LOW; not planned.

## Deferred (not rejected — needs its own plan / decision)

- **Ownership / access control on conversation read & delete.** GET
  `/conversations/:id` returns the user-side transcript (location queries = the
  PII `docs/errors.md` is otherwise careful to protect) and DELETE removes any
  conversation by id, with no identity check. Plan 002 adds throttling but
  deliberately does **not** close this. The real fix is an access-control model
  (secret token issued at creation, checked on read/delete) — a protocol-level
  change needing its own plan + an ADR + web-client changes. File as a
  follow-up before treating #3 as resolved.
- **HTTP route-layer test coverage / `startServer` agent-injection seam.**
  `startServer` hardcodes `createAgent` into the `ConversationStore`
  (`index.ts:353`) even though the store already accepts an `agentFactory`
  param (`conversations.ts:53`), so the POST/GET/DELETE routing, 404/409/403
  mapping, and SSE pipe are untested at the route level. Adding the seam +
  characterization tests is the prerequisite that would de-risk future route
  changes. Not scoped into this run.
